Pré-processamento (Pre-processing)

CSS por conta própria pode ser divertido, mas folhas de estilo estão ficando maiores, mais complexas e mais difíceis de manter. Este é o lugar onde um pré-processador pode ajudar. Sass permite que você use recursos que não existem em CSS ainda como variáveis​​, aninhamento, mixins, herança e outras coisas bacanas que tornam divertido escrever CSS novamente. Uma vez que você começar a mexer com Sass, terá o seu arquivo Sass(scss) pré processado e salvo como um arquivo CSS normal que você pode usar em seu web site.

Fragmentos (Partials)

Você pode criar arquivos Sass parciais que contêm pequenos trechos de CSS que você pode incluir em outros arquivos Sass. Esta é uma ótima maneira de modularizar seu CSS e ajudar a manter as coisas mais fáceis de gerenciar. A parcial é simplesmente um arquivo Sass nomeado com um sublinhado. Você pode nomeá-lo algo como _partial.scss. O sublinhado permite Sass sabe que o arquivo é apenas um arquivo parcial e que deve ser gerado em um arquivo CSS. Sass parciais são usados ​​com a diretiva @import.

Importar (Import)

CSS tem uma opção de importação que permite dividir seu CSS em partes menores. O único inconveniente é que cada vez que você usa @import no CSS ele cria um outro pedido HTTP. Sass constrói em cima do atual CSS, mas em vez de exigir uma nova solicitação HTTP, Sass irá importar o arquivo e combiná-lo com o arquivo principal para que você possa servir a um único arquivo CSS para o navegador da web.
